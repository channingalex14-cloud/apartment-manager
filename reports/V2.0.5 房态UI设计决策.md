# V2.0.5 房态 UI 设计决策

**版本**：V2.0.5
**更新**：2026-04-16
**文件**：`src/views/RoomList.vue`

---

## 一、设计目标

房态页面是系统核心页面，需让管理员一眼看清所有房间状态，并快速执行入住/退房/编辑等操作。

---

## 二、8 种房态定义

| 状态 | 中文名 | 语义 | 是否显示合同信息 |
|------|--------|------|----------------|
| `在租` | 在租 | 正常出租中 | ✅ |
| `新租` | 新租 | 刚入住不久 | ✅ |
| `空房` | 空房 | 空闲可出租 | ❌ |
| `员工` | 员工 | 员工住宿 | ❌ |
| `管理` | 管理 | 管理室 | ❌ |
| `违约` | 违约 | 违约中 | ❌ |
| `维修中` | 维修中 | 正在维修 | ❌ |
| `待清洁` | 待清洁 | 退房待清洁 | ❌ |

**不显示合同信息的 6 种状态**：`空房` `员工` `管理` `违约` `待清洁` `维修中`

```css
:root {
  --room-rented: #67c23a;       /* 在租 - 绿色 */
  --room-new: #409eff;           /* 新租 - 蓝色 */
  --room-vacant: #909399;        /* 空房 - 灰色 */
```

## 四、卡片设计

### 4.1 布局

- 使用 CSS Grid 布局，响应式
- 每个房间一张卡片
- 卡片高度固定，宽度自适应网格

### 4.2 卡片结构

```
┌─────────────────────────┐
│  101          [状态标签] │  ← 房号 + 状态
│                         │
│  张三                  │  ← 租客姓名（无合同则不显示）
│  2026-01 到 2026-12    │  ← 租期（无合同则不显示）
│                         │
│  ¥2000  ¥50   ¥2586   │  ← 租金/物业/账单总额
│                         │
│  [入住] [退房] [详情]   │  ← 操作按钮
└─────────────────────────┘
```

### 4.3 CSS 类方式

**已废弃方案**：内联 `:style="{ background: color }"`
**当前方案**：`:class="getStatusClass(room)"` + CSS 类

```vue
<div :class="['room-card', getStatusClass(room)]">
```

```css
.room-card.status-\u5728\u79c0 {
  background: var(--room-rented);
}
.room-card.status-\u7a7a\u623f {
  background: var(--room-vacant);
}
```

**优势**：
- 代码简洁，只需一行 `:class`
- 主题切换改 CSS 变量即可
- 天然支持 `:hover` 等伪类

## 五、空房透明度

空房状态下卡片添加半透明效果：

```css
.room-card.status-\u7a7a\u623f {
  background: var(--room-vacant);
  opacity: 0.6;  /* 半透明 */
}
```

## 八、Excel 导入状态映射

导入 Excel 时，原始状态文本需映射为枚举：

| Excel 原始值 | 映射结果 |
|-------------|---------|
| `在租` | `在租` |
| `新租` | `新租` |
| `空房` `空置` `空闲` | `空房` |
| `退房` `已退` `退租` | `待清洁` |
| `员工` | `员工` |
| `管理` | `管理` |
| `违约` | `违约` |
| `维修中` `维修` | `维修中` |
| `待清洁` `清洁` | `待清洁` |

**注意**：`退房` 映射为 `待清洁`，而非 `空房`。因为退房房间需要清洁后才能再次出租。
