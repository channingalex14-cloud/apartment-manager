### 缺陷 #1：诊断服务中 JoinError 被 to_string() 丢失结构化信息

**违反原则**：§6 错误处理 - 结构化错误转换

**文件**：`src-tauri/src/commands/diagnostic.rs` 第 12 行

**现状**：
```rust
.map_err(|e| { tracing::error!("spawn_blocking 任务失败: {}", e); e.to_string() })?;
```

**影响**：如果 spawn_blocking 任务 panic（如线程池耗尽），错误信息被截断为字符串 `"...cancelled"`，用户收到无意义的错误。

**修复方案**：
```rust
.map_err(|e| {
    tracing::error!("spawn_blocking 任务失败: {:?}", e);
    app_error_to_json_string(AppError::Business("后台任务执行失败，请稍后重试".to_string()))
})?;
```

### 缺陷 #2：认证服务中数据库错误暴露给用户

**违反原则**：§6 错误处理 - 错误信息不暴露内部实现

**文件**：`src-tauri/src/auth.rs` 第 240 行

**现状**：
```rust
.map_err(|e| AppError::PermissionDenied(e.to_string()))?;
```

**影响**：rusqlite 错误（如 "no such table: permissions"）会被暴露给用户，泄露数据库结构信息。

**修复方案**：
```rust
.map_err(|e| {
    tracing::error!("权限查询失败: {:?}", e);
    AppError::PermissionDenied("权限校验失败，请联系管理员".to_string())
})?;
```

### 缺陷 #3：queries.rs 中硬编码 LeaseStatus 状态值

**违反原则**：§3 状态机 - §1.3 禁止字符串硬编码

**文件**：`src-tauri/src/db/queries.rs` 第 377 行

**现状**：
```rust
"SELECT COUNT(*) FROM leases WHERE tenant_id = ? AND status IN ('生效', '待生效') AND is_deleted = 0",
```

**影响**：如果 LeaseStatus 枚举的字符串表示发生变化（如 "生效" → "Active"），此查询会失效。

**修复方案**：将 SQL 查询条件中的状态值参数化，由 Service 层传入正确的枚举字符串。

### 缺陷 #4：queries.rs 中硬编码 BillStatus 状态值

**违反原则**：§3 状态机 - §1.3 禁止字符串硬编码

**文件**：`src-tauri/src/db/queries.rs` 第 1097-1100 行

**现状**：
```rust
COALESCE(SUM(CASE WHEN status IN ('待缴费', '部分支付') THEN total_amount - actual_paid ELSE 0 END), 0) as total_pending, \
COUNT(*) as bill_count, \
SUM(CASE WHEN status IN ('待缴费', '部分支付') THEN 1 ELSE 0 END) as pending_count, \
```

**影响**：同上，枚举变化会导致统计结果错误。

### 缺陷 #6：bill_commands.rs 中序列化错误被静默忽略

**违反原则**：§1.4 前端禁止直接操作数据库 + 错误处理规范

**文件**：`src-tauri/src/commands/bill_commands.rs` 第 89, 101, 113, 125, 137 行

**现状**：
```rust
.map(|r| serde_json::to_string(&r).unwrap_or_default())
```

**影响**：如果序列化失败（如字段包含无效的 UTF-8），静默返回空 JSON `{}`，导致用户看到错误数据。

**修复方案**：
```rust
.map(|r| {
    serde_json::to_string(&r)
        .map_err(|e| app_error_to_json_string(AppError::Serialization(e)))?
});
```

### 缺陷 #7：queries.rs 中包含业务逻辑

**违反原则**：§1.4 前端禁止直接操作数据库（职责分离）

**文件**：`src-tauri/src/db/queries.rs` 第 841, 906, 1495, 1533 行

**现状**：
```rust
// 第 841 行 - query_bills_tx 函数
let count: i32 = match (param_year_month.as_ref(), ...) {
    (Some(ym), None, Some(rid), Some(st)) => { /* ... */ }
    (Some(ym), None, Some(rid), None) => { /* ... */ }
    // 共 12 种参数组合
};
```

**影响**：queries 层应只负责数据查询，业务逻辑（如根据参数选择不同的查询策略）应放在 Service 层。

**修复方案**：在 Service 层根据业务规则调用不同的查询函数。

### 缺陷 #8：多个 Service 使用事务外读取

**违反原则**：§7 事务模式 - TOCTOU 防护

**文件**：以下文件中的多个方法

| 文件 | 行号 |
|------|------|
| report_service.rs | 120, 173 |
| export_service.rs | 44 |
| document_service.rs | 67, 121 |
| reminder_service.rs | 60, 103 |

**现状**：
```rust
// report_service.rs 第 120 行
let conn = ctx.get_conn()?;
let mut stmt = conn.prepare("SELECT * FROM monthly_summary_cache WHERE year_month = ?")?;
```

**影响**：事务外读取可能导致读取到并发写入中的不一致数据。建议：对只读查询也使用事务包装以保持可重复读取（REPEATABLE READ 语义）。

**修复方案**：对于只读查询使用 `ctx.transaction(|tx| { ... })?` 包装。

### 缺陷 #9：Reports.vue 中手动 /100 转换未使用工具函数

**违反原则**：§1.1 金额单位 - 代码一致性

**文件**：`src/views/Reports.vue` 第 82-84 行

**现状**：
```typescript
const rentData = sortedReports.map(r => (r.rentTotal || 0) / 100);
const propertyData = sortedReports.map(r => (r.propertyTotal || 0) / 100);
const waterData = sortedReports.map(r => ((r.waterTotal || 0) + (r.electricTotal || 0)) / 100);
```

**修复方案**：
```typescript
import { toYuanNumber } from '@/utils/money';

const rentData = sortedReports.map(r => toYuanNumber(r.rentTotal || 0));
```

### 缺陷 #10：Reports.vue 中图表 Y 轴格式化使用手动计算

```typescript
const formatWanYuan = (v: number) => `¥${(v / 10000).toFixed(1)}万`;
```
