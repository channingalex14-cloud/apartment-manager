# 备份与恢复操作指南

> 适用版本：V2.0.5+
> 更新日期：2026-04-21

---

## 一、功能概述

系统提供数据库备份和JSON数据导出两种数据保护方式：

| 方式 | 说明 | 适用场景 |
|------|------|----------|
| **数据库备份** | 完整复制 SQLite 数据库文件 | 完整恢复，包括所有数据和配置 |
| **数据导出** | 导出 JSON 格式的结构化数据 | 数据分析、迁移到其他系统 |

---

## 二、数据库备份

### 2.1 手动备份

1. 打开 **设置** 页面
2. 找到 **数据备份** 卡片
3. 点击 **立即备份** 按钮
4. 确认提示框中的信息，点击确定

备份完成后会显示成功消息，备份文件自动保存到系统备份目录。

### 2.2 自动备份

1. 在 **数据备份** 卡片中
2. 开启 **启用自动备份** 开关
3. 设置 **保留份数**（默认 7 份）
4. 点击 **保存设置**

启用后，系统每日会自动执行一次数据库备份，并自动清理超过保留数量的旧备份。

### 2.3 备份文件位置

备份文件默认保存在数据库所在目录的 `backups` 子目录下：

```
{应用数据目录}/backups/
└── apartment_20260421_143025.db    # 格式：apartment_YYYYMMDD_HHMMSS.db
└── apartment_20260420_090000.db
└── ...
```

### 2.4 从备份恢复

> **警告：恢复操作会覆盖当前数据库，现有数据将丢失！**

1. 在 **备份记录** 表格中找到要恢复的备份
2. 点击该行的 **恢复** 按钮
3. 仔细阅读确认框中的警告信息
4. 确认无误后点击确定
5. 恢复完成后，根据提示重启应用

恢复前系统会自动创建紧急备份（`apartment.db.emergency`），以防恢复出现问题。

### 2.5 删除备份

如需删除不再需要的备份文件，点击备份记录表格中对应行的 **删除** 按钮即可。

## 三、数据导出

### 3.1 导出类型说明

| 类型 | 说明 | 是否支持年月筛选 |
|------|------|-----------------|
| 房间数据 | 所有房间的基本信息、状态、当前租客 | 否 |
| 租客数据 | 所有租客的联系方式 | 否 |
| 账单数据 | 所有账单明细及关联信息 | 是 |
| 缴费记录 | 所有缴费支付记录 | 是 |
| 年度汇总 | 各月账单统计汇总 | 否 |

### 3.2 导出操作步骤

1. 打开 **设置** 页面
2. 找到 **数据导出** 卡片
3. 选择 **导出类型**
4. 如需筛选特定年月，选择年月（仅账单和缴费记录支持）
5. 点击 **导出 JSON** 按钮
6. 在文件保存对话框中选择保存位置和文件名

### 3.3 导出文件格式

导出的 JSON 文件采用 UTF-8 编码，格式示例：

```json
{
  "exportType": "rooms",
  "exportTime": "2026-04-21 14:30:25",
  "recordCount": 12,
  "data": [
    {
      "id": 1,
      "room_number": "101",
      "building": "A栋",
      "status": "出租中",
      ...
    }
  ]
}
```

### 3.4 打开 JSON 文件

导出的 JSON 文件可用以下方式查看：

- **Excel**：直接用 Excel 打开，或通过 数据 → 从文本/CSV 导入
- **文本编辑器**：VS Code、Notepad++ 等均支持 UTF-8
- **JSON 查看器**：在线 JSON 格式化工具

## 四、备份设置说明

| 设置项 | 说明 | 默认值 |
|--------|------|--------|
| 启用自动备份 | 是否开启每日自动备份 | 关闭 |
| 保留份数 | 自动清理时保留的备份数量 | 7 份 |
| 备份目录 | 备份文件的保存位置 | `{应用数据目录}/backups` |

> **注意**：备份目录路径仅供参考，不建议手动修改。

---

## 五、注意事项

1. **定期备份**：建议至少每周执行一次手动备份，重要操作前（如批量导入）先备份
2. **保留策略**：建议保留至少 3 份以上备份，以便恢复不同时间点的数据
3. **备份验证**：恢复前建议先确认备份文件的创建时间是否符合预期
4. **紧急备份**：恢复操作会自动创建紧急备份，如恢复后出现问题可联系技术支持
5. **存储空间**：备份文件大小约为数据库文件的 1-2 倍，请确保存储空间充足

---

## 六、故障排除

### 备份失败

- **权限不足**：确保应用有写入备份目录的权限
- **磁盘空间不足**：清理磁盘空间后重试
- **数据库被占用**：关闭其他可能访问数据库的应用后重试

### 恢复后数据不一致

- 检查是否从正确的备份文件恢复
- 紧急备份文件位于数据库同目录下的 `apartment.db.emergency`
- 如问题持续，联系技术支持

### 导出文件打不开

- 确认文件下载/保存完整
- 使用支持 UTF-8 编码的编辑器打开
- 避免使用 Windows 记事本（建议使用 VS Code 或 Notepad++）
