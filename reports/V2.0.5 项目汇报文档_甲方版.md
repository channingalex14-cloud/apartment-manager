## 一、项目概述

### 1.1 项目定位

新逸公寓管理系统是一款专业的公寓租赁管理软件，专为中小型公寓房东和管理员设计。系统采用现代化技术栈开发，具备高性能、高可靠性和易用性的特点。

**核心价值**：
- 📊 **全面管理**：覆盖房间、租客、合同、账单、缴费、押金等全流程业务
- 💰 **财务清晰**：精确的账单生成、缴费记录、押金台账管理
- 📈 **数据可视化**：直观的房态展示、月度报表、统计图表
- 🔒 **数据安全**：本地存储，无云端依赖，数据完全自主掌控

## 二、核心功能模块

### 2.1 房态管理（核心功能）

**功能描述**：
- 可视化展示所有房间状态（8种状态）
- 支持按楼层、状态、房号筛选
- 双击房间卡片查看详情并打印收费通知单
- 房间状态变更自动记录日志

**房间状态定义**：

| 状态 | 颜色 | 允许操作 | 说明 |
|------|------|---------|------|
| 空房 | 🟢 绿色 | 入住、手动切换 | 可出租房间 |
| 在租 | 🔵 蓝色 | 退房、违约标记 | 正常出租中 |
| 新租 | 🔵 青色 | 退房、违约标记 | 新入住未满月 |
| 员工 | 🟣 紫色 | 手动切换 | 员工宿舍 |
| 管理 | ⚪ 灰色 | 手动切换 | 管理自用 |
| 违约 | 🔴 红色 | 退房、恢复 | 欠租违约状态 |
| 待清洁 | 🟡 黄色 | 入住、手动切换 | 退房待清理 |
| 维修中 | 🟠 橙色 | 手动切换 | 设施维修中 |

**状态机约束**：
- ✅ 允许手动切换：空房 ↔ 维修中/管理/员工/待清洁
- ❌ 禁止手动切换：在租/新租/违约 → 必须通过退房或违约流程

### 2.2 合同管理

**核心流程**：
```
创建合同(草稿) → 激活合同 → 入住 → [正常退房/违约处理] → 归档
```

**合同状态流转**：
```
草稿 → 生效中 ⟷ 违约中 → 待结算 → 已退房 → 已归档
  ↓
已作废
```

**入住流程**（check_in）：
1. 验证房间状态（必须为空房或待清洁）
2. 验证合同状态（必须为生效中）
3. 收取押金（可选）
4. 更新房间状态为在租/新租
5. 记录操作日志

**退房流程**（check_out）：
1. 验证房间状态（必须为在租/新租/违约）
2. 结算押金（退还或没收）
3. 更新房间状态为待清洁
4. 更新合同状态为已退房
5. 记录退房日期和原因

### 2.3 账单管理

**账单生成流程**：
```
抄表录入 → meter_readings表 → 生成账单 → monthly_bills表
```

**账单构成**：
- 房租（支持半月折算）
- 物业费（按月固定）
- 水费（用量 × 单价，最低消费）
- 电费（用量 × 单价，最低消费）
- 管理费（用量 × 单价，最低消费）
- 维修费（可选）
- 杂费（可选）
- 往期欠款（自动累计）

**账单状态流转**：
```
待缴费 ──[确认支付]──→ 已支付（终态）
  ├──[部分支付]──→ 部分支付 ──[确认支付]──→ 已支付
  │                └──[作废]──→ 已作废
  └──[作废]──→ 已作废 ──[重新生成]──→ 待缴费（UPDATE恢复）
```

**账单类型**：
- 正常：完整月账单
- 首月免水电：新入住首月
- 半月结算：月中入住/退房
- 末月结算：退房最后一个月
- 月中退房结算：中途退房

### 2.4 缴费管理

**缴费方式**：
- 微信支付
- 支付宝
- 银行卡转账
- 现金
- 商家码
- 押金抵扣
- 混合支付（多种方式组合）

**混合支付示例**：
```
账单金额: 2586元
支付方式: 微信 1500元 + 现金 1000元 + 押金抵扣 86元
```

### 2.5 押金台账

**押金交易类型**：
- 收取：入住时收取押金
- 退还：退房时退还押金
- 没收：违约时没收押金
- 抵扣：抵扣欠费

**押金状态**：
- 未收取
- 已收取
- 部分收取
- 退还
- 没收

### 2.6 报表统计

**月度报表内容**：
- 房间统计：总数、在租、空房、违约等
- 收费统计：应收总额、实收总额、收缴率
- 费用明细：房租、水电、物业等分项
- 趋势分析：多月份对比图表

**首页仪表盘**：
- 实时统计卡片
- 收租进度环形图
- 月度趋势折线图
- 待缴费账单列表

## 三、数据库设计

### 3.1 核心表结构

**业务表（15张）**：

| 表名 | 中文名 | 说明 | 核心字段 |
|------|--------|------|---------|
| rooms | 房间表 | 107间房源信息 | 房号、楼层、状态、租金 |
| tenants | 租客表 | 租客基本信息 | 姓名、手机、紧急联系人 |
| leases | 合同表 | 租赁合同信息 | 房间、租客、起止日期、押金 |
| monthly_bills | 月度账单表 | 每月收费账单 | 房间、月份、费用明细、状态 |
| payments | 缴费记录表 | 缴费流水 | 账单、金额、方式、日期 |
| deposit_ledger | 押金台账表 | 押金流水账 | 合同、类型、金额、余额 |
| meter_readings | 抄表记录表 | 水电表读数 | 房间、年月、水表、电表 |
| room_status_log | 房间状态日志 | 状态变更记录 | 房间、原状态、新状态、触发类型 |
| tenant_history | 租客历史表 | 租客事件记录 | 租客、事件类型、关联房间 |
| system_config | 系统配置表 | 系统参数 | 配置键、配置值 |
| notice_templates | 通知模板表 | 收费通知模板 | 模板名称、内容 |
| documents | 文档表 | 合同文档等 | 文件名、类型、路径 |
| reminders | 提醒表 | 事务提醒 | 标题、日期、状态 |
| operation_logs | 操作日志表 | 操作审计 | 操作类型、详情、时间 |
| monthly_summary_cache | 月度汇总缓存 | 报表缓存 | 月份、统计数据 |

### 3.2 金额存储规范

**核心原则**：所有金额以**整数分**存储，避免浮点精度问题。

| 存储值 | 显示值 | 说明 |
|--------|--------|------|
| 200000 | ¥2000.00 | 月租金 |
| 600 | ¥6.00 | 水费单价（元/方） |
| 73 | ¥0.73 | 电费单价（元/度） |
| 258600 | ¥2586.00 | 账单总额 |

### 3.3 索引设计

**关键索引**：
- rooms: room_number, floor, status
- leases: room_id, tenant_id, status, start_date, end_date
- monthly_bills: year_month, room_id, status
- payments: bill_id, room_id, payment_date
- deposit_ledger: lease_id, transaction_type

### 3.4 软删除策略

**设计原则**：所有业务表包含 `is_deleted` 字段，删除操作设置标志位而非物理删除。

```sql
-- 查询规范：必须过滤已删除
SELECT * FROM rooms WHERE is_deleted = 0;

-- 删除操作：设置标志位
UPDATE rooms SET is_deleted = 1 WHERE id = ?;
```

## 四、业务流程详解

### 4.1 新租客入住流程

```
1. 创建租客信息
   ↓
2. 创建合同（草稿状态）
   - 选择房间
   - 填写租期
   - 设置租金、押金
   ↓
3. 激活合同
   ↓
4. 执行入住（check_in）
   - 验证房间状态
   - 收取押金
   - 更新房间状态
   - 记录日志
   ↓
5. 首月抄表录入
   ↓
6. 生成首月账单
```

### 4.2 月度收费流程

```
每月25日：
1. 抄表录入
   - 录入各房间水表读数
   - 录入各房间电表读数
   - 系统自动计算用量
   ↓
2. 生成账单
   - 批量生成所有在租房间账单
   - 自动计算各项费用
   - 自动累计往期欠款
   ↓
3. 打印通知单
   - 双击房间卡片
   - 查看账单详情
   - 打印收费通知单
   ↓
4. 缴费记录
   - 选择缴费方式
   - 记录缴费金额
   - 支持混合支付
   ↓
5. 账单状态更新
   - 待缴费 → 已支付
   - 或 待缴费 → 部分支付
```

### 4.3 退房结算流程

```
1. 执行退房（check_out）
   - 验证房间状态
   - 结算押金
   - 更新房间状态为待清洁
   ↓
2. 生成末月账单（如需）
   - 按实际天数折算房租
   - 结算水电费用
   ↓
3. 押金处理
   - 退还剩余押金
   - 或抵扣欠费
   - 或没收违约金
   ↓
4. 房间清洁
   - 手动更新状态：待清洁 → 空房
   ↓
5. 下一位租客入住
```

## 九、项目亮点

### 9.1 技术亮点

1. **现代化技术栈**
   - Tauri 2.0：比 Electron 更轻量，内存占用仅 30MB
   - Rust 后端：高性能、内存安全、无 GC 停顿
   - Vue 3 Composition API：更好的代码组织和复用

2. **严谨的架构设计**
