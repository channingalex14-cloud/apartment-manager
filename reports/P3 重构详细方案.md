## 一、import_service.rs 重构方案

### 1.1 职责边界

**从 import_commands.rs 提取的业务逻辑**：

| 函数 | 职责 | 是否需要事务 |
|------|------|-------------|
| `parse_excel_file()` | Excel 解析（外部依赖） | 否 |
| `get_or_create_tenant()` | 租客查找/创建 | 否（只读/插入） |
| `get_room_id()` | 房间查询 | 否 |
| `insert_bill()` | 账单插入 | 否 |
| `insert_payment()` | 缴费记录插入 | 否 |
| `update_room_status()` | 房间状态更新+日志 | **是** |
| `normalize_status()` | 状态映射 | 否 |
| `delete_existing_bill_for_room_month()` | 软删除旧账单 | **是** |

### 1.2 新增 Service 文件结构

```rust
// src-tauri/src/services/import_service.rs

use crate::db::{queries, AppContext, HasConnection};
use crate::errors::{AppError, Result};
use crate::models::{BillStatus, RoomStatus, BillRow};

pub struct ImportService;

impl ImportService {
    /// 导入月度账单（主要入口）
    pub fn import_monthly_bills<C: HasConnection>(
        &self,
        ctx: &C,
        file_path: &str,
        year_month: &str,
        operator: Option<&str>,
    ) -> Result<ImportResult> {
        // 1. 解析 Excel
        let rows = self.parse_excel(file_path)?;

        // 2. 遍历导入（每条记录一个事务）
        for row in rows {
            ctx.transaction(|tx| {
                self.import_single_bill(tx, &row, year_month, operator)
            })?;
        }

        Ok(result)
    }

    /// 解析 Excel 文件
    fn parse_excel(&self, path: &str) -> Result<Vec<BillRow>>;

    /// 导入单条账单（事务内）
    fn import_single_bill(
        &self,
        tx: &Transaction,
        row: &BillRow,
        year_month: &str,
        operator: Option<&str>,
    ) -> Result<()> {
        // 1. 删除旧账单
        self.delete_existing_bill(tx, room_id, year_month)?;

        // 2. 租客处理
        let tenant_id = self.get_or_create_tenant(tx, &row.tenant_name, &row.phone)?;

        // 3. 状态映射
        let room_status = self.normalize_status(&row.room_status);

        // 4. 在租房间：创建/更新合同
        if matches!(room_status, RoomStatus::Rented | RoomStatus::NewRented | ...) {
            self.ensure_lease_exists(tx, room_id, tenant_id, row)?;
        }

        // 5. 插入账单
        self.insert_bill(tx, row, room_id, year_month)?;

        // 6. 插入缴费记录（如已缴费）
        if row.payment_status == "已交" {
            self.insert_payment(tx, bill_id, room_id, row)?;
        }

        // 7. 更新房间状态（含日志）
        self.update_room_status(tx, room_id, room_status, &row.room_type, operator)?;
    }
}
```

### 1.3 依赖分析

**import_service.rs 需要依赖**：
- `queries.rs`：已有 `get_lease_by_id_tx`、`update_room_tx` 等
- `lease_service.rs`：用于 `ensure_lease_exists` 调用现有合同服务
- `bill_service.rs`：用于调用现有账单服务
- `payment_service.rs`：用于调用现有缴费服务


## 二、diagnostic_service.rs 重构方案

### 2.1 职责边界

**从 diagnostic.rs 提取的业务逻辑**：

| 函数 | 职责 | 是否需要事务 |
|------|------|-------------|
| `fix_meter_fees()` | 修复账单水电费 | **是**（批量更新） |
| `fix_management_rooms()` | 修复管理房间关联 | **是**（批量更新） |

### 2.2 新增 Service 文件结构

```rust
// src-tauri/src/services/diagnostic_service.rs

use crate::db::{queries, AppContext, HasConnection};
use crate::errors::{AppError, Result};
use crate::models::{LeaseStatus, DepositStatus};
use rusqlite::params;

pub struct DiagnosticService;

impl DiagnosticService {
    /// 修复账单水电费
    ///
    /// 遍历所有账单，根据 meter_readings 重新计算水电费和管理费
    pub fn fix_meter_fees<C: HasConnection>(&self, ctx: &C) -> Result<DiagnosticReport> {
        ctx.transaction(|tx| {
            // 1. 获取单价配置
            let water_unit_price = self.get_unit_price(tx, "水费单价")?;
            let electric_unit_price = self.get_unit_price(tx, "电费单价")?;
            let management_unit_price = self.get_unit_price(tx, "管理费单价")?;

            // 2. 获取所有账单
            let bills = queries::list_bills_for_fix(tx)?;

            // 3. 遍历修复
            let mut fixed_count = 0;
            for bill in bills {
                self.fix_single_bill(tx, &bill, water_unit_price, electric_unit_price, management_unit_price)?;
                fixed_count += 1;
            }

            Ok(DiagnosticReport { fixed_count, .. })
        })
    }

    /// 修复单个账单的水电费
    fn fix_single_bill(
        &self,
        tx: &Transaction,
        bill: &Bill,
        water_unit_price: i64,
        electric_unit_price: i64,
        management_unit_price: i64,
    ) -> Result<()> {
        // 获取前后读数
        let (prev_water, prev_electric) = self.get_prev_readings(tx, bill.room_id, &bill.year_month)?;
        let (curr_water, curr_electric) = self.get_curr_readings(tx, bill.room_id, &bill.year_month)?;

        // 计算用量和费用
        let water_usage = (curr_water - prev_water).max(0);
        let electric_usage = (curr_electric - prev_electric).max(0);

        let water_fee = water_usage.checked_mul(water_unit_price).ok_or_else(|| AppError::business("水费溢出"))?;
        let electric_fee = electric_usage.checked_mul(electric_unit_price).ok_or_else(|| AppError::business("电费溢出"))?;
        let management_fee = electric_usage.checked_mul(management_unit_price).ok_or_else(|| AppError::business("管理费溢出"))?;

        // 计算总费用（租金+物业+水电+管理+其他）
        let total = bill.rent_fee + bill.property_fee + water_fee + electric_fee + management_fee + bill.repair_fee;

        // 更新账单
        queries::update_bill_fees_tx(tx, bill.id, water_fee, electric_fee, management_fee, total)?;

        Ok(())
    }

    /// 修复管理房间租客关联
    pub fn fix_management_rooms<C: HasConnection>(
        &self,
        ctx: &C,
        fixes: Vec<RoomTenantFix>,
    ) -> Result<DiagnosticReport> {
        ctx.transaction(|tx| {
            let mut fixed_count = 0;
            for fix in fixes {
                self.fix_room_tenant(tx, &fix)?;
                fixed_count += 1;
            }
            Ok(DiagnosticReport { fixed_count, .. })
        })
    }
}
```

```rust
pub struct ExportService;

    pub fn export_data<C: HasConnection>(
```

```rust
/// 导出房间查询
pub fn export_rooms_query(conn: &Connection) -> Result<Vec<serde_json::Value>> {
    let mut stmt = conn.prepare(
        "SELECT r.id, r.room_number, r.floor, r.building, r.room_type,
                r.base_rent, r.property_fee, r.deposit, r.status,
                t.name as tenant_name, t.phone as tenant_phone
         FROM rooms r
         LEFT JOIN leases l ON r.id = l.room_id AND l.status = 'active'
         LEFT JOIN tenants t ON l.tenant_id = t.id
         WHERE r.is_deleted = 0
         ORDER BY r.building, r.room_number"
    )?;

    let rows = stmt.query_map([], |row| {
        Ok(json!({
            "id": row.get::<_, i64>(0)?,
            "room_number": row.get::<_, String>(1)?,
            // ... 其他字段
        }))
    })?;

    rows.collect::<Result<Vec<_>>>()
}
```

## 六、重构原则

### 6.1 保持向后兼容

- 不要改变命令的输入输出格式
- 不要改变命令的签名（参数名、参数类型）
- 测试用例应该仍然能通过

### 6.2 渐进式重构

每个 Phase 分 3 步：
1. **创建新 Service 文件**，复制原逻辑
2. **修改 Commands**，委托给新 Service
3. **删除原辅助函数**（确认测试通过后）

### 6.3 错误处理一致

所有 Service 方法返回 `Result<T, AppError>`，由 Commands 层统一转换为 `Result<T, String>`。
