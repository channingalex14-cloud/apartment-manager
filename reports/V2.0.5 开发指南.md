## 一、核心开发原则

### 1.1 金额单位：统一使用 INTEGER（分）

```rust
// ❌ 错误：使用浮点数
let rent: f64 = 2000.0; // 元

// ✅ 正确：使用整数（分）
let rent: i64 = 200000; // 分 = 2000元
```

**转换规则**：
- 前端输入元 → 后端存分：`amount * 100`
- 后端查分 → 前端显元：`amount / 100`

```typescript
// 前端 utils/money.ts

/**
 * 元转分（避免浮点陷阱，使用字符串解析）
 * 19.9 * 100 = 1989.999...  ❌
 * 字符串解析: "19.9" → 1990  ✅
 */
export function toCent(yuan: number | string): number {
  const valueStr = typeof yuan === "string" ? yuan : String(yuan);
  const isNegative = valueStr.startsWith("-");
  const absValueStr = isNegative ? valueStr.slice(1) : valueStr;
  const parts = absValueStr.split(".");
  const intPart = parseInt(parts[0] || "0", 10) * 100;
  const decPart = parts[1] ? parts[1].padEnd(2, "0").slice(0, 2) : "00";
  const result = intPart + parseInt(decPart, 10);
  return isNegative ? -result : result;
}

export function toYuanNumber(fen: number): number {
  return Math.round((fen / 100) * 100) / 100;
}
```

### 1.2 日期格式：统一 `YYYY-MM-DD`

- 所有日期字段存储为 TEXT
- 使用 `date('now')` 或 `datetime('now')` 获取当前日期
- Rust 层使用 `chrono::Local::now().format("%Y-%m-%d")`

### 1.3 状态机：应用层全权负责

数据库**不**包含任何触发器执行状态变更。状态机逻辑全部在 Service 层实现。

**必须使用 `LeaseStatus` / `BillStatus` 枚举进行状态校验，禁止字符串硬编码**。

```rust
// ✅ 正确：使用枚举校验
let current_status = LeaseStatus::from_str(&lease.status)?;
if !current_status.can_transition_to(&LeaseStatus::Active) {
    return Err(AppError::InvalidStatus(...));
}

// ❌ 错误：字符串硬编码
if lease.status != "生效中" { ... }
```

### 1.4 前端禁止直接操作数据库

所有数据库访问必须通过 `invoke()` 调用 Rust Commands。

```typescript
// ✅ 正确
import { leaseService } from '@/services/lease.service';
await leaseService.checkIn(req);

// ❌ 错误：直接操作数据库
```

### 1.5 表单验证（schemas）

使用 Zod 进行表单验证，采用 `lazySchema` 模式避免循环依赖。

```typescript
// ✅ 正确：使用 lazySchema 避免循环依赖
import { validateRoomForm } from '@/schemas';
const result = validateRoomForm(data);
if (!result.success) {
  ElMessage.error(result.errors.join('\n'));
}
```

### 1.6 全局事件总线（events）

跨组件通信使用事件总线模式，基于 Signal 实现。

```typescript
// 监听事件
import { roomEvents } from '@/events';
const unsub = roomEvents.openBillPrint.subscribe(({ roomId, billId }) => {
  // 处理事件
});
onUnmounted(unsub);

// 触发事件
roomEvents.openBillPrint.emit({ roomId: 1, billId: 100 });
```

**已有事件**：
- `roomEvents.openBillPrint` - 打开收费通知单
- `roomEvents.roomStatusChanged` - 房间状态变更
- `roomEvents.billPaid` - 账单已支付

## 三、状态机枚举

### 3.1 LeaseStatus 枚举定义

```rust
// src-tauri/src/models/lease.rs

pub enum LeaseStatus {
    Draft,         // 草稿
    Active,        // 生效中
    Violation,     // 违约中
    PendingSettle, // 待结算
    CheckedOut,   // 已退房
    Archived,      // 已归档
    Cancelled,     // 已作废
}

impl LeaseStatus {
    /// 从字符串转换为枚举
    pub fn from_str(s: &str) -> Result<Self> {
        match s {
            "草稿" => Ok(Self::Draft),
            "生效中" => Ok(Self::Active),
            "违约中" => Ok(Self::Violation),
            "待结算" => Ok(Self::PendingSettle),
            "已退房" => Ok(Self::CheckedOut),
            "已归档" => Ok(Self::Archived),
            "已作废" => Ok(Self::Cancelled),
            _ => Err(AppError::InvalidStatus(format!("未知合同状态: {}", s))),
        }
    }

    /// 获取枚举对应的字符串
    pub fn as_str(&self) -> &'static str { ... }

    /// 检查是否可以流转到目标状态
    pub fn can_transition_to(&self, next: &LeaseStatus) -> bool {
        matches!(
            (self, next),
            (Self::Draft, Self::Active)
                | (Self::Active, Self::Violation)
                | (Self::Active, Self::PendingSettle)
                | (Self::Violation, Self::PendingSettle)
                | (Self::Violation, Self::Active)      // 违约恢复
                | (Self::PendingSettle, Self::CheckedOut)
                | (Self::CheckedOut, Self::Archived)
                | (Self::Draft, Self::Cancelled)
        )
    }
}
```

### 3.2 Lease 状态流转图

```
草稿 → 生效中 ⟷ 违约中 → 待结算 → 已退房 → 已归档
  ↓              ↑
  已作废         └── 违约恢复（Violation → Active）
```

// src-tauri/src/models/bill.rs

pub enum BillStatus {
    Pending,      // 待缴费
    Partial,      // 部分支付
    Paid,         // 已支付（终态）
    Voided,       // 已作废（终态）
}

impl BillStatus {
    pub fn from_str(s: &str) -> Result<Self> { ... }

    /// 检查是否可以流转到目标状态
    pub fn can_transition_to(&self, next: &BillStatus) -> bool {
        matches!(
            (self, next),
            // 待缴费 → 已支付 / 部分支付 / 已作废
            (Self::Pending, Self::Paid)
                | (Self::Pending, Self::Partial)
                | (Self::Pending, Self::Voided)
            // 部分支付 → 已支付 / 已作废
                | (Self::Partial, Self::Paid)
                | (Self::Partial, Self::Voided)
            // 已作废 → 待缴费（重新生成）
                | (Self::Voided, Self::Pending)
        )
    }
}
```

```

  ├──[部分支付]──→ 部分支付 ──[确认支付]──→ 已支付（终态）
  │                └──[作废]──→ 已作废（终态）
  └──[作废]──→ 已作废（终态）──[重新生成]──→ 待缴费（UPDATE 恢复）

pub enum RoomStatus {
    Vacant,      // 空房
    Rented,      // 在租
    NewRented,   // 新租
    Staff,       // 员工
    Management,   // 管理
    Violation,   // 违约

        matches!(self, Self::Rented | Self::NewRented | Self::Violation)
    }

## 四、Service 层核心实现 (Rust)

### 4.1 入住（Check-In）

**事务边界**：所有操作（读取 + 写入）必须在同一事务中执行，防止 TOCTOU。

```rust
pub fn check_in(&self, ctx: &AppContext, req: &CheckInRequest) -> Result<LeaseResponse> {
    // ✅ 读取和写入都在同一事务内（TOCTOU 防护）
    ctx.transaction(|tx| {
        // 1. 验证房间（在事务内读取）
        let room = queries::get_room_by_id_tx(tx, req.room_id)?
            .ok_or_else(|| AppError::not_found("房间", req.room_id))?;

        // 2. 验证房间状态（使用 RoomStatus 枚举，禁止字符串硬编码）
        let room_status = RoomStatus::from_str(&room.status)?;
        if !room_status.allows_check_in() {
            return Ok(LeaseResponse::error(&format!(
                "房间状态不允许入住: {}", room.status
            )));
        }

        // 3. 验证合同（在事务内读取）
        let lease = queries::get_lease_by_id_tx(tx, req.lease_id)?
            .ok_or_else(|| AppError::not_found("合同", req.lease_id))?;

        // 4. 验证合同状态（使用 LeaseStatus 枚举）
        let current_status = LeaseStatus::from_str(&lease.status)?;
        if current_status != LeaseStatus::Active {
            return Ok(LeaseResponse::error(&format!(
                "合同状态不允许入住: {}", lease.status
            )));
        }

        // 5. 收取押金（调用 deposit_service）
        if lease.deposit > 0 {
            deposit_service.receive_deposit(ctx, req.lease_id, req.room_id, lease.deposit)?;
        }

        // 6. 更新房间状态为"新租"
        tx.execute("UPDATE rooms SET status = '新租', updated_at = datetime('now') WHERE id = ?", [req.room_id])?;

        // 7. 更新合同入住日期
        tx.execute("UPDATE leases SET move_in_date = ?, updated_at = datetime('now') WHERE id = ?", params![req.move_in_date, req.lease_id])?;

        // 8. 记录房间状态变更日志
        tx.execute("INSERT INTO room_status_log ...", params![...])?;

        // 9. 记录租客历史
        tx.execute("INSERT INTO tenant_history ...", params![...])?;

        Ok(())
    })?;

    Ok(LeaseResponse::success_with_id(req.lease_id))
}
```

### 4.2 退房（Check-Out）

**押金处理**：正常退房退还押金，违约退房没收押金。

```rust
pub fn check_out(&self, ctx: &AppContext, req: &CheckOutRequest) -> Result<LeaseResponse> {
    ctx.transaction(|tx| {
        // 1. 验证合同（在事务内读取）
        let lease = queries::get_lease_by_id_tx(tx, req.lease_id)?
            .ok_or_else(|| AppError::not_found("合同", req.lease_id))?;

        // 2. 使用状态机校验
        let current_status = LeaseStatus::from_str(&lease.status)?;
        if !current_status.can_transition_to(&LeaseStatus::PendingSettle) {
            return Ok(LeaseResponse::error(&format!(
                "合同状态不允许退房: {}", lease.status
            )));
        }

        // 3. 处理押金（退还/没收）
        if lease.deposit_balance > 0 {
            match req.reason.as_str() {
                "正常退房" => {
                    // 退还押金
                    tx.execute("INSERT INTO deposit_ledger (..., '退还', ...) ...", params![...])?;
                    tx.execute("UPDATE leases SET deposit_balance = 0, deposit_status = '退还' ...", params![...])?;
                }
                "违约退房" => {
                    // 没收押金
                    tx.execute("INSERT INTO deposit_ledger (..., '没收', ...) ...", params![...])?;
                    tx.execute("UPDATE leases SET deposit_balance = 0, deposit_status = '没收' ...", params![...])?;
                }
                _ => {
                    // 其他原因，默认退还
                    tx.execute("INSERT INTO deposit_ledger (..., '退还', ...) ...", params![...])?;
                    tx.execute("UPDATE leases SET deposit_balance = 0, deposit_status = '退还' ...", params![...])?;
                }
            }
        }

        // 4. 更新合同状态为"待结算"
        tx.execute("UPDATE leases SET status = '待结算', move_out_date = ?, termination_reason = ? ...", params![...])?;

        // 5. 更新房间状态为"待清洁"
        tx.execute("UPDATE rooms SET status = '待清洁' ...", params![...])?;

        Ok(())
    })?;

    Ok(LeaseResponse::success_with_id(req.lease_id))
}
```

### 4.3 账单生成（Bill Service）

**半月折算**：新租首月按实际入住天数折算房租。

```rust
// 获取半月天数
fn get_half_month_days(move_in_date: &str, year_month: &str) -> i64 {
    let move_in = NaiveDate::parse_from_str(move_in_date, "%Y-%m-%d").unwrap();
    let year: i32 = year_month[0..4].parse().unwrap();
    let month: u32 = year_month[5..7].parse().unwrap();
    let last_day = date_of_last_day(year, month);
    let days_in_month = (last_day - first_day).num_days() + 1;
    let move_in_day = move_in.signed_duration_since(first_day).num_days() + 1;

    if move_in_day >= days_in_month / 2 + 1 {
        days_in_month - move_in_day + 1  // 下半月入住
    } else {
        move_in_day  // 上半月入住
    }
}

// 折算半月房租
fn calculate_half_month_fee(monthly_rent: i64, days: i64, total_days: i64) -> i64 {
    (monthly_rent * days + total_days - 1) / total_days  // 向上取整
}
```

## 七、事务模式

### 7.1 事务边界原则

**一个业务操作 = 一个事务**。

```rust
// ✅ 正确：所有操作在同一事务
ctx.transaction(|tx| {
    tx.execute("INSERT INTO deposit_ledger ...", params![...])?;
    tx.execute("UPDATE rooms SET ...", params![...])?;
    tx.execute("UPDATE leases SET ...", params![...])?;
    tx.execute("INSERT INTO room_status_log ...", params![...])?;
    Ok(())
})?;
```

### 7.2 锁顺序规则

**防止死锁**：多表操作时，按固定顺序访问表。

```
rooms → tenants → leases → monthly_bills → payments → deposit_ledger
```

### 7.3 TOCTOU 防护

读取和写入必须在同一事务中：

```rust
// ✅ 正确：事务内读取
ctx.transaction(|tx| {
    let bill = tx.query_row("SELECT ... WHERE id = ?", [bill_id])?;
    tx.execute("UPDATE monthly_bills SET ...", [...])?;
    Ok(())
})?;

// ❌ 错误：事务外读取
let bill = conn.query_row("SELECT ...", [...])?;  // 读取
ctx.transaction(|tx| { ... })?;                    // 写入
```

## 十三、注意事项

1. **禁止字符串硬编码状态** - 必须使用枚举校验
2. **禁止在 SQL 中拼接用户输入** - 使用参数化查询 (`params![]`)
3. **金额计算统一用分** - 避免浮点精度问题
4. **状态变更必须记录日志** - 不可跳过 `room_status_log`
5. **押金操作必须记录台账** - 不可只改余额
6. **所有业务操作使用事务** - 保持数据一致性
7. **软删除不物理删除** - 只设置 `is_deleted = 1`
8. **账单生成后不修改单价** - 使用快照值
9. **耗时操作用 `spawn_blocking`** - 避免阻塞主线程
10. **禁止 `unwrap()`** - 使用 `?` 或 `if let` 替代
11. **退房押金区分处理** - 正常退房退还，违约退房没收

### 17.2 require_permission! 宏

权限校验辅助宏，用于细粒度权限控制：

```rust
macro_rules! require_permission {
    ($conn:expr, $role:expr, $resource:expr, $permission:expr, $action:expr) => {{
        let required = $action;
        match check_permission($conn, $role, $resource, $permission, &required) {
            Ok(true) => {}
            Ok(false) => return Err(AppError::PermissionDenied(...)),
            Err(e) => return Err(e),
        }
    }};
}
```

### 17.3 前端 Token 管理

```typescript
// auth.service.ts
export const authService = {
  // 登录
  async login(request: LoginRequest): Promise<LoginResponse> {
    const response = await callCommand<LoginResponse>("login", { request });
    if (response.success && response.token) {
      localStorage.setItem("auth_token", response.token);
      localStorage.setItem("auth_user", JSON.stringify(response.user));
    }
    return response;
  },

  // 获取当前用户
  async getCurrentUser(): Promise<UserInfo | null> {
    const savedToken = localStorage.getItem("auth_token");
    if (!savedToken) return null;
    // 调用后端验证
  },

  // 登出
  async logout(): Promise<boolean> {
    localStorage.removeItem("auth_token");
    localStorage.removeItem("auth_user");
    return true;
  }
};
```
