# V2.0.5 新逸公寓管理系统架构概览

**版本**：V2.0.5
**更新**：2026-04-27
**技术栈**：Tauri 2.0 + Vue 3 + TypeScript + Rust + SQLite

---

## 一、系统定位

新逸公寓管理系统是一款面向中小型公寓的桌面应用，用于管理房间状态、租客、合同、账单、缴费和押金等核心业务。

**目标用户**：公寓房东/管理员
**运行平台**：Windows（通过 Tauri 打包）
**数据存储**：本地 SQLite（无云端依赖）

## 五、数据流：一次账单生成

```
1. 前端: BillGenerate.vue → billService.generate(req)
                              ↓ invoke("generate_monthly_bills")
2. Rust: bill_commands::generate_monthly_bills
                              ↓ ctx.transaction(|tx|)
3. Service: bill_service::generate_monthly_bills
   - 查询所有在租/新租房间
   - 对每个房间读取 meter_readings（抄表记录）
   - 计算水电用量 × 单价
   - 半月折算（如适用）
   - INSERT monthly_bills
4. 返回: { success_count, skipped_count, errors[] }
```

## 六、关键设计决策

### 6.1 金额存储：分而非元
所有金额在数据库中以整数分存储，避免浮点精度问题。
前端显示时 /100 转为元。

### 6.2 状态机集中管理
LeaseStatus / BillStatus / RoomStatus 枚举在 Service 层校验，禁止字符串硬编码。

### 6.3 TOCTOU 防护
所有读取和写入必须在同一事务内。使用 `_tx` 后缀函数在事务内读取。

### 6.4 软删除规范
所有业务表含 `is_deleted` 字段，删除操作设置标志位而非物理删除。

### 6.5 月度汇总缓存
`monthly_summary_cache` 表缓存月度统计数据，避免每次页面加载重新聚合。

## 八、Phase 4 权限系统

### 8.1 核心组件

| 组件 | 文件 | 说明 |
|------|------|------|
| 认证服务 | `auth_service.ts` | Token 管理、会话持久化 |
| 认证命令 | `auth_commands.rs` | login/logout/get_current_user/list_users/check_permission |
| 权限宏 | `require_permission!` | 权限校验辅助宏 |
| 用户模型 | `users/permissions 表` | 用户和权限数据模型 |

### 8.2 认证流程

```
1. 用户输入账号密码 → Login.vue
2. 调用 login 命令 → auth_commands::login
3. 验证密码（bcrypt） → auth_service::login
4. 生成 Token → 返回 LoginResponse
5. Token 存储到 localStorage
6. 路由守卫检查 Token → 无效跳转登录页
```

| 用户名 | 密码 | 角色 |
|--------|------|------|
| admin | admin123 | 管理员 |
